- 주문·체결 인터페이스는 존재하지 않는다.

## 4. 결정과 대안

### DEC-001: 단계형 분석 파이프라인

- 상태: `PROPOSED`
- 동인: `DRV-001`~`DRV-005`
- 결정: 분석을 입력 검증 및 현재 구성, 데이터 정규화, 대형주 판정, 예측·백테스트, 목표 포트폴리오 산출, 결과 조립의 명시적 단계로 실행한다. 각 단계는 성공 결과 또는 구조화된 실패를 반환하며 실패한 결과를 후속 단계의 정상 입력으로 사용하지 않는다.
- 대안: 하나의 결합된 계산 모듈. 초기 구현은 단순하지만 실패 격리, 누수 검사 및 추적성이 약해져 채택하지 않는다.
- 근거: 단계별 기준 시점과 버전을 고정하고 검증할 수 있다.
- 결과: 중간 계약과 단계별 시험이 필요하지만 계산 책임과 실패 원인이 분명해진다.
- 영향 컴포넌트: `CMP-002`~`CMP-006`
- 검증: `AC-029`~`AC-037`의 고정 데이터, 정책 충돌, 누수 및 상태 주입 시험
- 추적 링크: `FR-022`~`FR-025`, `NFR-012`~`NFR-015`, `QA-012`~`QA-015`
- 미해결 위험: 입력 정책 값은 `OPEN-ARCH-001`~`OPEN-ARCH-008`이 결정될 때까지 일부 단계 실행을 차단한다.

### DEC-002: 기준 시점이 포함된 표준 분석 데이터 계약

- 상태: `PROPOSED`
- 동인: `DRV-002`, `DRV-003`, `DRV-005`
- 결정: 외부 데이터는 종목 식별자, 값, 관측 시각, 해당 값이 이용 가능해진 시각, 출처 버전 및 상태를 포함한 내부 계약으로 정규화한다. 모든 판정과 모의 의사결정은 명시된 기준 시점으로 이 계약을 조회한다.
- 대안: 원본 공급자 형식을 분석 단계가 직접 사용. 공급자 결합과 시점 의미의 혼동 때문에 채택하지 않는다.
- 근거: 데이터 출처 선택을 열어 둔 채 누수 검사와 상태 표시를 일관되게 수행한다.
- 결과: 어댑터 구현과 계약 검증이 필요하다.
- 영향 컴포넌트: `CMP-003`, `CMP-004`, `CMP-006`
- 검증: 미래 이용 가능 시각을 가진 음성 표본에서 위반 탐지, 누락·오래됨·오류 상태 주입
- 추적 링크: `FR-023`, `FR-024`, `NFR-013`~`NFR-015`, `QA-013`~`QA-015`
- 미해결 위험: 출처와 최신성 기준은 `OPEN-ARCH-003`, `OPEN-ARCH-008`에 의존한다.

### DEC-003: 실행 스냅샷 기반 결과 재현

- 상태: `PROPOSED`
- 동인: `DRV-001`, `DRV-004`, `DRV-005`
- 결정: 각 분석 실행은 입력 스냅샷, 기준 시점, 데이터·모델·정책·백테스트 버전, 생성 시각, 단계별 상태 및 최종 결과를 하나의 실행 ID로 연결한다. 실행 상태는 `요청됨` → `실행 중` → (`실패` 또는 `완료`)만 허용한다. `완료`는 필수 메타데이터, 모든 필수 단계의 종결 상태, 최종 결과와 완료 시각이 같은 실행 ID에 일관되게 기록된 뒤에만 표시한다. 재실행은 동일 스냅샷과 버전 참조를 입력으로 사용하되 실패한 실행을 변경하지 않고 새 실행 ID를 생성한다.
- 대안: 최종 화면 값만 저장. 결과 차이의 원인을 검증할 수 없어 채택하지 않는다.
- 근거: 개발 단계에서 필요한 최소 출처 추적과 반복 비교를 제공한다.
- 결과: 재실행에 필요한 버전 자산은 실행 ID가 참조 가능한 동안 접근 가능해야 한다. 같은 실행 ID에 대한 동일 내용의 상태 쓰기는 멱등적으로 처리하고, 이미 기록된 내용과 충돌하는 쓰기나 종결 상태 변경은 거절한다. 장기 보존 정책은 범위 밖이다.
- 영향 컴포넌트: `CMP-002`, `CMP-006`
- 검증: 저장된 실행을 반복하고 필수 메타데이터 완전성과 수치 허용오차 비교
- 추적 링크: `NFR-012`, `NFR-014`, `QA-012`, `QA-014`
- 미해결 위험: 허용오차는 `OPEN-ARCH-007`에 의존한다.

### DEC-004: 표현 모델을 통한 웹 UX 분리

- 상태: `PROPOSED`
- 동인: `DRV-005`, `DRV-006`
- 결정: 웹 UX는 계산 모듈의 내부 객체 대신 상태, 기준 시각, 근거, 현재·목표 구성, 예측·백테스트 및 위험을 포함한 읽기 전용 표현 모델을 소비한다. 상태는 색상 외 텍스트 의미를 반드시 포함한다.
- 대안: 계산 결과 객체를 화면에 직접 결합. 화면 검증과 분석 변경의 독립성이 낮아 채택하지 않는다.
- 근거: 부분 실패와 비정상 상태를 정상 결과와 혼동하지 않고 다양한 화면에서 같은 의미로 표현한다.
- 결과: 표현 모델 매핑 시험과 반응형·키보드 시험이 필요하다.
- 영향 컴포넌트: `CMP-001`, `CMP-002`, `INT-001`, `INT-005`
- 검증: 상태 주입, 키보드 탐색, 결정된 화면 크기의 겹침·잘림 점검
- 추적 링크: `FR-026`, `NFR-015`, `NFR-016`, `QA-015`, `QA-016`
- 미해결 위험: 지원 화면 범위는 `OPEN-ARCH-009`에 의존한다.

## 6. 인터페이스와 핵심 흐름

### INT-001: 웹 분석 요청

- 생산자와 소비자: `CMP-001` → `CMP-002`
- 계약: 브라우저에서 받은 모든 값을 비신뢰 입력으로 간주한다. `CMP-001`은 허용된 필드만 수용하고 종목 식별자의 허용 문자·길이, 양수 수량의 형식·범위, 시각·버전 참조 형식, 보유 항목 개수와 전체 요청 크기를 구현 계획에서 정한 한도에 따라 검증한다. `CMP-002`는 동일 계약을 경계에서 재확인하며, 성공 시 실행 ID, 실패 시 원인·필드·허용 형식을 포함한 구조화된 오류를 반환한다.
- 인증·권한 경계: 사용자/브라우저 → `CMP-001` → `CMP-002`는 비신뢰 입력 경계다. 개발 단계 애플리케이션 인증·권한은 두지 않으며, 개발자가 통제하는 로컬 또는 격리 환경 내부 호출로 한정한다.
- 실패·복구: 잘못된 입력은 실행하지 않고 원인과 필드를 반환한다. 동일 요청의 자동 재시도는 하지 않는다.
- 호환성: 필수 필드와 의미를 버전으로 식별하며 알 수 없는 필수 버전은 거절한다.
- 시간 제한·재시도: 구체 값은 구현 계획에서 정한다. 요청 제한 시 실행 상태 조회가 가능한 실행 ID를 반환해야 한다.
- 관찰성: 실행 ID, 요청 시각, 검증 결과 및 단계 상태를 기록하되 상품 운영 로그는 범위 밖이다.

### INT-002: 외부 시장·재무 데이터 조회

- 생산자와 소비자: 외부 데이터 출처 → `CMP-003`
- 계약: 원본 종목 자료를 조회하고 내부적으로 종목 ID, 값, 관측 시각, 이용 가능 시각, 출처 버전, 상태로 변환한다.
- 인증·권한 경계: 공급자별 접근 방식은 `OPEN-ARCH-003` 이후 결정한다. 자격 증명 설계는 현재 범위에 포함하지 않는다.
- 실패·복구: 누락, 지연, 오류를 정상값으로 대체하지 않고 구조화된 상태로 반환한다. 제한적 재시도 여부는 출처 선택 후 정한다.
- 호환성: 외부 형식 변화는 어댑터 안에서 흡수하고 내부 계약 변경은 명시적 버전 상승을 요구한다.
- 관찰성: 출처, 조회 기준 시점, 수신 시각, 변환 상태를 실행 ID에 연결한다.

### INT-003: 적격성·예측·백테스트 실행

- 생산자와 소비자: `CMP-002` → `CMP-004`
- 계약: 기준 시점이 고정된 표준 데이터, 대형주 정책 버전, 예측 기간 및 모델·백테스트 버전을 입력받아 종목별 적격 근거, 예측, 불확실성, 성능 지표 또는 구조화된 실패를 반환한다.
- 인증·권한 경계: 시스템 내부 인터페이스이며 별도 인증 없음.
- 실패·복구: 필수 정책·데이터가 없거나 누수 검사에 실패하면 해당 결과를 생성하지 않는다. 자동으로 다른 정책·모델을 선택하지 않는다.
- 호환성: 입력·결과 스키마와 모델 계약 버전을 확인한다.
- 관찰성: 버전, 기준 시점, 누수 검사 결과, 소요 단계와 실패 원인을 기록한다.

### INT-004: 포트폴리오 계산

- 생산자와 소비자: `CMP-002` → `CMP-005`
- 계약: 현재 보유 가치, 적격 후보 분석과 포트폴리오 정책을 입력받아 합계 100%의 목표 비중, 차이, 후보 행동, 위험·집중도 또는 정책 충돌을 반환한다.
- 인증·권한 경계: 시스템 내부 인터페이스이며 별도 인증 없음.
- 실패·복구: 해가 없거나 제약이 충돌하면 목표안을 반환하지 않고 충돌한 정책을 식별한다. 제약을 임의 완화하지 않는다.
- 호환성: 산식·정책 버전을 결과에 포함한다.
- 관찰성: 입력 버전, 적용 제약, 계산 상태와 검증 합계를 실행 ID에 연결한다.

### INT-005: 분석 결과 표현 모델 조회

- 생산자와 소비자: `CMP-002`/`CMP-006` → `CMP-001`
- 계약: 실행 ID에 대해 현재·목표 구성, 예측·백테스트, 위험·리밸런싱 근거, 출처 버전, 기준 시각 및 영역별 `최신|오래됨|누락|오류` 상태를 반환한다.
- 인증·권한 경계: 내부 실행 증거에서 웹 UX로 나가는 출력 경계다. 애플리케이션 인증은 없으며 결과와 보유 기반 정보는 개발자가 통제하는 로컬 또는 격리된 개발 환경에서만 노출한다. 외부 공개 환경으로의 노출은 이 계약 밖이며 별도 요구사항 승인이 필요하다.
- 실패·복구: 일부 결과가 없으면 가용 부분과 영향 영역을 함께 반환한다. 정상·최신 상태로 승격하지 않는다.
- 호환성: 표현 모델 버전을 제공하며 웹 UX는 지원하지 않는 필수 버전을 거절한다.
- 관찰성: 실행 ID, 조회 시각과 결과 상태를 개발 진단에 남긴다.

### INT-006: 실행 증거 저장·조회

- 생산자와 소비자: `CMP-002` ↔ `CMP-006`
- 계약: 실행 ID별 입력 스냅샷, 기준 시점, 버전 참조, 단계 상태, 결과 및 생성 시각을 일관된 실행 기록으로 저장하고 조회한다. 상태는 `요청됨`, `실행 중`, `실패`, `완료`이며 `완료`는 필수 입력·버전 참조, 모든 필수 단계의 종결 상태, 최종 결과 및 완료 시각이 존재하고 완전성 검사를 통과한 경우에만 기록한다.
- 인증·권한 경계: 시스템 내부 인터페이스이며 별도 인증 없음.
- 실패·복구: 필수 증거 저장 실패 시 `완료`로 표시하지 않는다. 종결 표식이 없거나 필수 필드·단계 기록이 누락된 실행은 조회 시 `부분 기록`으로 탐지하여 `실패`로 전이 가능한 진단 대상으로 반환한다. 같은 실행 ID에 대한 동일 내용의 반복 쓰기는 멱등적으로 성공시키고, 기존 값과 충돌하는 쓰기 또는 `실패`·`완료` 종결 상태의 변경은 거절한다. 일시적 저장 실패는 같은 실행 ID로 미확정 쓰기를 재시도할 수 있으나, 분석 재실행은 기존 기록을 덮어쓰지 않고 새 실행 ID를 생성한다. 중단 지점부터 계산을 재개하는 기능은 현재 보장하지 않는다.
- 호환성: 저장 형식 버전과 읽기 호환 정책을 명시한다.
- 관찰성: 저장 성공·실패와 누락 필드를 실행 ID로 진단한다.

핵심 정상 흐름은 `INT-001` → `INT-002` → `INT-003` → `INT-004` → `INT-006` → `INT-005` 순이다. 현재 구성 계산은 입력 검증 후 가능하지만, `OPEN-ARCH-001`~`OPEN-ARCH-006`에 필요한 값이 없으면 대형주 판정 이후 흐름은 `차단됨` 상태로 종료한다.

## 8. 배포와 운영 모델

- 단일 개발·검증 환경에서 웹 UX와 분석 서비스를 실행하는 논리 모델만 정의한다. 프로세스 수, 호스트 수, 컨테이너, 클라우드, 데이터베이스 및 네트워크 토폴로지는 결정하지 않는다.
- 외부에서 공개 접근하는 상용 서비스는 가정하지 않는다. 실행 환경은 개발자가 통제하는 로컬 또는 격리된 개발 환경이다.
- 분석 실행은 실행 ID로 `요청됨|실행 중|실패|완료` 상태와 부분 기록 진단을 확인할 수 있어야 하며 웹 요청 수명과 장시간 계산 수명을 결합하지 않는다. 구체적인 비동기 구현 방식은 열어 둔다.
- 개발 진단은 실행 ID, 단계, 기준 시각, 버전, 상태 및 실패 원인을 포함한다. 서비스 수준 목표, 경보, 당직, 자동 확장, 재해 복구 목표는 상품화 범위이므로 정의하지 않는다.
- 외부 데이터가 없어도 고정 검증 데이터로 `QA-012`~`QA-016`을 실행할 수 있어야 한다.

## 10. 보안, 개인정보 및 준수

- 애플리케이션 인증, 권한, 세션, 전송 보호, 마스킹, 보존 정책 및 관할권 고지는 승인된 요구사항의 제외 범위이므로 설계하지 않는다.
- 이 제외는 상용 배포에 충분한 보안을 의미하지 않는다. 외부 공개 또는 실제 사용자 운영 전에 별도 요구사항과 위협 분석이 필요하다.
- 개발 단계에서는 합성·비식별 보유 데이터 사용, 개발 환경 접근 제한, 비밀값의 소스·실행 증거 미포함을 운영 전제로 둔다. 이는 상품 기능이 아니라 개발 안전 수칙이다.
- 브라우저 입력은 개발 환경에서도 비신뢰 데이터로 검증하며 분석 결과는 개발자가 통제하는 환경 안에서만 노출한다. 이는 상품용 인증·권한을 추가하지 않고 입력·출력 신뢰 경계를 명확히 하는 최소 안전 전술이다.
- 시스템에는 주문 제출 경계가 없으며 외부 데이터 어댑터를 주문·브로커 인터페이스로 확장하지 않는다.

## 12. 아키텍처 결정 기록

### ADR-001: 분석 단계와 책임 경계

- 상태: `PROPOSED`
- 결정: `DEC-001`의 단계형 파이프라인과 `CMP-003`~`CMP-005`의 책임 분리를 채택한다.
- 동인: `DRV-001`~`DRV-005`
- 대안과 근거: 결합 모듈은 시점 검증과 실패 격리가 어려워 제외한다.
- 결과: 내부 계약 수가 늘지만 단계별 검증과 교체가 가능하다.
- 영향: `CMP-002`~`CMP-006`, `INT-002`~`INT-006`
- 요구사항/품질 추적: `FR-022`~`FR-025`, `NFR-012`~`NFR-015`, `QA-012`~`QA-015`
- 검증: 단계별 계약 시험과 종단 간 고정 데이터 시험
- 미해결 위험: `RISK-001`~`RISK-003`

### ADR-002: 내부 시점·상태 데이터 계약

- 상태: `PROPOSED`
- 결정: `DEC-002`의 표준 계약을 모든 분석 입력의 경계로 사용한다.
- 동인: `DRV-002`, `DRV-003`, `DRV-005`
- 대안과 근거: 공급자 원본 직접 사용은 시점 의미와 상태 처리를 분산시켜 제외한다.
- 결과: 공급자 변경은 `CMP-003`에 국한되며 데이터 계약 버전 관리가 필요하다.
- 영향: `CMP-003`, `CMP-004`, `INT-002`, `INT-003`, `DATA-002`
- 요구사항/품질 추적: `FR-023`, `FR-024`, `NFR-013`, `NFR-015`, `QA-013`, `QA-015`
- 검증: 계약 적합성, 누수 음성 시험, 상태 주입 시험
- 미해결 위험: `RISK-001`, `OPEN-ARCH-003`, `OPEN-ARCH-008`

### ADR-003: 실행 단위 출처 추적

- 상태: `PROPOSED`
- 결정: `DEC-003`의 실행 스냅샷을 재현과 결과 조회의 기준으로 삼고 `요청됨` → `실행 중` → (`실패` 또는 `완료`) 상태 전이, 완료 전 완전성 검사, 부분 기록 탐지, 동일 실행 ID의 멱등·충돌 거절 규칙을 적용한다. 저장 재시도는 미확정 쓰기에 한정하고 분석 재실행은 새 실행 ID를 사용한다.
- 동인: `DRV-001`, `DRV-004`, `DRV-005`
- 대안과 근거: 최종값만 저장하면 버전과 입력을 복구할 수 없어 제외한다.
- 결과: 실행 증거 저장이 분석 완료 조건의 일부가 된다.
- 영향: `CMP-002`, `CMP-006`, `INT-006`, `DATA-001`~`DATA-005`
- 요구사항/품질 추적: `NFR-012`, `NFR-014`, `QA-012`, `QA-014`
- 검증: 필수 메타데이터 완전성, 중단·부분 기록 탐지, 같은 실행 ID의 동일·충돌 쓰기 및 새 실행 ID 반복 실행 비교
- 미해결 위험: `RISK-003`, `OPEN-ARCH-007`

### ADR-004: 웹 표현 모델 경계

- 상태: `PROPOSED`
- 결정: `DEC-004`의 읽기 전용 표현 모델을 웹 UX의 유일한 결과 계약으로 사용한다.
- 동인: `DRV-005`, `DRV-006`
- 대안과 근거: 분석 객체 직접 노출은 상태 의미와 UX를 내부 계산에 결합시켜 제외한다.
- 결과: 표현 매핑 계층이 추가되며 상태·접근성 시험을 독립 수행할 수 있다.
- 영향: `CMP-001`, `CMP-002`, `INT-005`
- 요구사항/품질 추적: `FR-026`, `NFR-015`, `NFR-016`, `QA-015`, `QA-016`
- 검증: 상태 주입 및 화면 크기별 키보드·시각 점검
- 미해결 위험: `OPEN-ARCH-008`, `OPEN-ARCH-009`
